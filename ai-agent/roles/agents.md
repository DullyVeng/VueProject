# 三个 Agent：

1. code-explorer
   - 作用：专职查找项目中的相关代码、函数、类、文件路径
   - 注意：只输出定位信息和关键片段，不写实现代码
   - 使用场景：任务拆分时定位逻辑、查找复用、定位 bug

2. code-architect
   - 作用：架构设计与方案评估
   - 注意：输出模块拆分、接口设计和潜在风险分析，不写实现代码
   - 使用场景：任务拆分阶段、设计确认、需求临时调整

3. code-reviewer
   - 作用：代码审查
   - 注意：关注可维护性、边界条件、潜在 bug、回归风险
   - 使用场景：每个 Task 完成后或提测前的系统性 CR
